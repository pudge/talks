Issue #1: Newlines

  • \n is a line separator

    • It is a logical construct, not a physical character
    • May vary between platforms or IO types

  • DOS has two main IO modes

    • In text files, "\n" eq "\015\012"
    • seek() and tell() problems, because \n is now two bytes long
    • In binary files and sockets, "\n" eq "\012" (like Unix)
    • Use binmode() on binary files
