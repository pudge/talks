"Text" files

  • Perl now handles CRs in source files
    • In program text, CRs treated like other whitespace
    • In string literals, CRs ignored if paired with \n

  • Files that scripts may read as data are on their own
    • Paragraph mode ($/ = undef) will fail if file is not read in "text" mode

  • C compiler may not like CRs in XS

  • Do not put literal CRs in source files
