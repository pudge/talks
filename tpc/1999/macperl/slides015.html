Object Specifier Records

  • This is where Apple events get messy
  • An AE object consists of four parts: class, container, data form, and data (and type).
  • In Mac::Glue, order determines containers, class and data are specified, and data form and type are guessed (or can be stated explicitly)
  • Example: 
        $folder = $finder->obj( file => 'Foo', folder => 'Bar', disk => 2 );
    $finder->open($folder);
  • "disk" object contains "folder" object which contains "file" object
  • Data for "folder" and "file" objects are type typeChar and form formName, since they are text. Data for "disk" is type typeInteger and form formAbsoluteIndex. The obj_form function allows specifying other forms and types.
