I'm Glad You Asked

  • An application that can be controlled via Apple events is called scriptable
  • Most Mac files have a resource fork, and scriptable applications have an aete resource, containing all the Apple event interface data for that application
  • Included data are "human" names for events, classes, parameters, properties, etc.
  • Drop an application on "gluemac" droplet to create glue -- and documentation -- for that application
  • Then just start scripting it
